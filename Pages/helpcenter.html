ENGINE DATABASE / 001

About RPG Maker VX Ace

SSIA: Vanguard Chronicles is built with RPG Maker VX Ace. This engine is lightweight, Windows-focused, and suitable for 2D story-driven RPG experiences with tactical systems, dialogue scenes, maps, events, and custom scripts.

  • Best played on Windows using the official game folder provided by TSBC Games Division.
  • Do not delete core files such as Game.exe, Game.ini, Audio, Graphics, Data, or System folders.
  • Keep the game inside a normal folder path, not inside restricted system folders.
  • If the game cannot open, try extracting the game again and running it from the extracted folder.
ENGINE DATABASE / 002

About RPG Maker MV

RPG Maker MV is a newer RPG Maker engine that uses JavaScript and supports broader deployment targets. If SSIA: Vanguard Chronicles ever uses MV-related builds or experiments, it may behave differently from the VX Ace version.

  • MV projects usually use different save systems compared to VX Ace.
  • VX Ace save files should not be assumed compatible with MV unless TSBC officially confirms it.
  • MV may support more modern deployment options, but performance depends on device, browser/runtime, and plugins.
  • Always follow the version notes included in the official TSBC release post.
SAVE SUPPORT / 003

Save Data Recovery and Backup

Save data is important for continuing your Vanguard Chronicles progress. Before updating, moving, deleting, or reinstalling the game, make a backup of your save files first.

  • Open your current SSIA: Vanguard Chronicles game folder.
  • Look for save files such as Save01.rvdata2, Save02.rvdata2, or similar VX Ace save files.
  • Copy the save files into a backup folder, external drive, cloud folder, or another safe location.
  • If a save becomes corrupted, replace it with your latest working backup.
  • Do not rename save files unless you understand the save slot order.
UPDATE SUPPORT / 004

How To Copy Paste Save Data into Newer Version of the Game

When TSBC releases a newer version, you can usually continue your progress by copying your save files from the old game folder into the new game folder, as long as the update remains compatible with previous saves.

  • Download and extract the newest SSIA: Vanguard Chronicles version.
  • Open the old game folder and find your SaveXX.rvdata2 files.
  • Copy the save files from the old game folder.
  • Paste them into the new game folder, near Game.exe, unless the release notes say otherwise.
  • Launch the new version and check whether your save slot appears correctly.
  • Keep the old game folder until you confirm the new version works properly.
PERFORMANCE SUPPORT / 005

My PC is at Low Specification

SSIA: Vanguard Chronicles is designed to be playable on modest Windows hardware, but low specification PCs may still experience slow loading, stutter, or audio delay depending on storage speed, background apps, and system health.

  • Close heavy apps such as browsers, launchers, screen recorders, and editing software before playing.
  • Play while your laptop is plugged in if your device lowers performance on battery mode.
  • Extract the game fully before running it; do not play directly from a compressed archive.
  • Move the game to internal storage or SSD if loading feels slow.
  • Restart your PC if the game becomes unusually slow after long usage.
  • Keep enough free storage so Windows can run smoothly.
FIELD REPORT / 006

Bug Reports

If you find a bug, glitch, crash, typo, map issue, softlock, missing dialogue, or broken event, send a clear report so TSBC can investigate it faster.

  • Include your game version, for example: Version_DD_MM_YY.
  • Explain where the bug happened: map name, scene, chapter, or quest step.
  • Describe what you were doing before the bug appeared.
  • Attach screenshots or video if possible.
  • Mention whether the bug happens once or can be repeated.
  • Do not delete your save backup until the issue is solved.
Important Notice: Always read the official release notes before updating SSIA: Vanguard Chronicles. Some updates may include new content, balance changes, event changes, file structure changes, or save compatibility notes.
